Ideas Principales

Módulo Flexbox

Tiene como objetivo proporcionar una manera más eficiente de diseñar, alinear y distribuir el espacio entre los elementos en un contenedor, incluso cuando el tamaño es desconocido o dinámico. La idea principal es otorgar al contenedor la capacidad de modificar ancho, alto y el orden de los elementos para llenar mejor el espacio. Un contenedor flexible expande los elementos o los reduce para evitar el desbordamiento. Lo más importante es que su diseño es independiente de la dirección en lugar de los diseños regulares.
Flex-direction: Esta propiedad establece el eje principal en el que se colocarán los elementos dentro del contenedor flexible.
Valores:

    row: Los elementos se organizan de izquierda a derecha (valor predeterminado).
    row-reverse: Los elementos se organizan de derecha a izquierda.
    column: Los elementos se organizan de arriba hacia abajo en forma de columnas.
    column-reverse: Los elementos se organizan en columnas, pero de abajo hacia arriba.
Flex-wrap Define si los elementos dentro del contenedor deben mantenerse en una sola línea o pueden envolver en varias líneas según sea necesario.
Valores:
    nowrap: Todos los elementos intentarán mantenerse en una sola línea, aunque esto pueda causar desbordamiento (valor predeterminado).
    wrap: Los elementos se distribuyen en múltiples filas o columnas si es necesario, evitando el desbordamiento.
    wrap-reverse: Similar a wrap, pero los elementos se colocan en sentido inverso, envolviendo hacia arriba en lugar de hacia abajo.
Flex-flow Es una abreviatura para combinar las propiedades flex-direction y flex-wrap. Permite definir simultáneamente la dirección principal y si los elementos deben envolverse.
Justify-content Controla la alineación de los elementos a lo largo del eje principal (horizontal para row o vertical para column). Esta propiedad es útil para distribuir los elementos cuando hay más espacio disponible que el necesario.
Valores:
    flex-start: Los elementos se alinean al inicio del contenedor.
    flex-end: Los elementos se alinean al final del contenedor.
    center: Los elementos se centran en el contenedor.
    space-between: El primer y último elemento se alinean en los extremos del contenedor, mientras que los demás se distribuyen uniformemente.
    space-around: Los elementos se distribuyen uniformemente con espacios iguales alrededor de cada uno.
Align-items Similar a justify-content, pero aplicado en el eje transversal (perpendicular al principal). Define cómo se alinean los elementos dentro del contenedor en el eje vertical cuando flex-direction es row.
Valores:
    stretch: Los elementos se estiran para llenar el espacio vertical disponible (valor predeterminado).
    flex-start, flex-end, center: Similar a los valores de justify-content, pero aplicados al eje transversal.
    baseline: Los elementos se alinean a lo largo de su línea base.
Align-content Esta propiedad afecta la alineación de las filas o columnas en contenedores flexibles con múltiples líneas (cuando se usa flex-wrap: wrap). No tiene efecto si solo hay una línea de elementos.
Valores:
    flex-start, flex-end, center, space-between, space-around: Funciona de manera similar a justify-content, pero para las líneas dentro del contenedor.

Notas

Conocer las propiedades: Es importante tener claro el funcionamiento y los valores predeterminados de cada propiedad para evitar errores. Una mala configuración puede generar problemas como desbordamientos o una alineación incorrecta de los elementos.
Flexbox es unidireccional: Solo gestiona una dirección a la vez, ya sea horizontal o vertical, lo que significa que no es la mejor opción para layouts complejos de múltiples ejes. Para esos casos, Grid Layout es más adecuado.

Bitácora

Resumen

La idea clave detrás de Flexbox es que el contenedor flexible puede modificar las propiedades de ancho, alto y el orden de sus elementos internos para adaptarse de manera óptima al espacio disponible, algo que los diseños tradicionales no permiten de forma sencilla.

Propiedad Width

Propiedad max-width: la imagen se puede reducir, pero esta no se amplía más allá de sus dimensiones originales. Esto nos puede ayudar a que la imagen no pierda la calidad por aumentar sus dimensiones, sin embargo, el problema es que quedan espacios en blanco cuando llegue a sus dimensiones máximas.
Imágenes de fondo: las imágenes de fondo también pueden aumentar y reducir su tamaño. Con la propiedad background-size: 100% 100% la imagen se ajustara al elemento contenedor, sin embargo la imagen deberá adaptarse sin distorsionarse.
Backgorund-size: cover: mantiene la relación de aspecto pero la imagen se puede recortar cuando se aumenta o disminuye el tamaño del dispositivo. Con background-position se puede ajustar el enfoque que hará background-size: cover.
Por medio de los media queries podemos cambiar la imagen cuando se llegue al max-width especificado, siempre y cuando este como una imagen de fondo. Sin embargo dentro del HTML también se puede cambiar. El elemento picture, ajustara el tamaño de la imagen, lo que se hace es colocar mediante los atributos source srcste seguido de media:”(max-width)”. Luego la siguiente imagen con su respectivo media:”(max-width)” y por último la imagen original solo con img src=.

Notas

Max-width: Reduce la imagen sin aumentar más allá de su tamaño original. Mantiene la calidad de la imagen, pero puede dejar espacios en blanco en el contenedor.
Imágenes de Fondo: background-size: 100% 100%: Ajusta la imagen al contenedor, pero puede distorsionarla.
background-size: cover: Mantiene la relación de aspecto; puede recortar la imagen.
background-position: Ajusta el enfoque de la imagen cuando se usa background-size: cover.
Media Queries: Cambian la imagen de fondo al alcanzar un max-width especificado.
Elemento picture: Permite cambiar imágenes según el tamaño de pantalla usando source con media: "(max-width)".

Bitácora

Resumen

Max-width: Ayuda a mantener la calidad de la imagen y evita que se amplíe más allá de sus dimensiones originales.
background-size: Controla cómo se muestra la imagen de fondo, con opciones para cubrir y ajustar sin distorsión.
Media Queries y picture: Herramientas fundamentales para un diseño responsivo, permitiendo cambios en las imágenes según el tamaño del dispositivo.

Gráficos Vectoriales Escalables (SVG)

SVG es un lenguaje de marcado para describir imágenes y aplicaciones de gráficos bidimensionales y un conjunto de interfaces. Se convirtió en una recomendación W3C siendo un lenguaje libre de derechos de autor de uso generalizado.
La versión 2 se encuentra en desarrollo e integrara con HTML, CSS y DOM. Los archivos SVG son gráficos vectoriales bidimensionales tanto estáticos como animados en formato XML y se puede utilizar con imágenes como formas. Son compactos y ofrecen gráficos de buena calidad, está basado en XML y se puede usar para crear gráficos web que respondan a acciones del usuario.

Notas

    SVG (Scalable Vector Graphics) es un lenguaje de marcado para describir gráficos bidimensionales.
    Se convirtió en estándar del W3C y es un lenguaje libre de derechos de autor.
    La versión 2 de SVG está en desarrollo e integrará mejor con HTML, CSS y DOM.
